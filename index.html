Production MLOps Feature Store — Real-Time Serving & Drift Detection

What It Solves

You train a fraud model. It works. Six months later it silently degrades because transaction patterns shifted. This system prevents that. Dual-store: Redis (HGETALL returns all 34 features in one round-trip at 0.37ms P99) for sub-ms online serving, PostgreSQL with point-in-time correct time-travel queries for offline training — eliminating training-serving skew by construction. Every feature snapshot is stamped with valid_from; training queries return the value the model would have seen at prediction time, not current values.

Original Findings
✓
PSI + KL divergence detected gradual drift 19× faster than mean±2σ monitoring (1h vs 19h). Seasonal shift: 6× faster. Sudden shift: 2× faster. KL catches histogram shape changes while the mean is still within 2 standard deviations.
✓
P99 0.37ms across 1,000 reads of 34 features each — well inside the <10ms gate. End-to-end prediction P99: 5.6ms across 10,000 requests.
✓
False positive rate 2.35% across 1,700 feature checks (target <5%) — achieved by sampling from the exact baseline histogram, not a Gaussian approximation.

04Multi-AgentResearch

Collaborative Multi-Agent GenAI System

In-progress research — Agent A and Agent B independently generate responses, then a separate critique agent decides whether to merge both outputs or select one. Evaluated on novelty, practicality, and quality.

▾
In Progress

Investigating which multi-agent collaboration architecture produces superior outputs vs a single-agent baseline. The core question: does independent parallel generation with a dedicated merge/select judge outperform iterative critique-refine? Each architecture is systematically implemented and evaluated.

Architectures Under Study
A
Generate-Critique-Refine (GCR): Agent A generates → a dedicated Critic agent evaluates for accuracy and completeness → Agent A refines. Convergence-based loop termination.
B
Parallel Generate + Merge: Agent A and Agent B independently generate full responses → a separate Critique agent decides whether to merge both outputs into a stronger answer or select the better one outright.
C
Single Agent Baseline: Standard single-model output used as the evaluation baseline across all criteria.
Evaluation Criteria (3)
1
Novelty: does the multi-agent output surface insights or perspectives absent from the single-agent response?
2
Practicality: is the response directly actionable — concrete steps, not abstract advice?
3
Quality: overall accuracy, coherence, and completeness scored by LLM-as-judge against ground truth.
